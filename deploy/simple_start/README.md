# 最简单启动方式

现场人员优先使用项目根目录下的两个入口：

```text
启动AI服务.bat
检查AI服务.bat
```

本目录 `deploy/simple_start/` 保存实际脚本逻辑，根目录的 bat 只是快捷入口。

## 启动服务

在项目根目录双击：

```text
启动AI服务.bat
```

窗口打开后不要关闭。窗口存在时，AI API 服务就在运行。

上位机调用地址：

```text
http://127.0.0.1:8000/api/v1/dose/recommend/simple
```

状态检查地址：

```text
http://127.0.0.1:8000/api/v1/ready
```

## 检查服务

在项目根目录双击：

```text
检查AI服务.bat
```

如果显示 FAIL，请先双击 `启动AI服务.bat`。

## 停止服务

关闭 `启动AI服务.bat` 打开的窗口，就等于停止 AI 服务。

内部还保留了维护脚本：

```text
deploy\simple_start\stop_ai_service.bat
```

它只给维护人员在窗口找不到、服务卡住、8000 端口被占用时强制释放端口使用。现场人员日常不需要点这个脚本。

注意：维护停止脚本会停止监听 `8000` 端口的 Python/uvicorn 服务。如果同一台电脑上还有其他 8000 端口服务，不要使用维护停止脚本。

## 现场默认配置

启动脚本默认使用 CPU 模式：

```text
MODEL_DEVICE=cpu
USE_BACKUP=true
CPU_THREADS=4
```

这表示现场优先使用 XGBoost 备用模型，减少无 GPU 电脑上的启动时间。

## 限制

这个方案最简单，但不是后台守护服务：

- 电脑重启后需要重新双击启动。
- 关闭窗口后服务就停止。
- 服务异常退出后不会自动重启。

正式长期运行时，再考虑 Windows 服务或开机自启方案。
